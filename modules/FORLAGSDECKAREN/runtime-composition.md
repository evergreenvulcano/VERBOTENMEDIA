# Runtime Composition Layer

## Role

Runtime composition turns the admitted source packet into bounded drafting surfaces for FĂ¶rlagsdeckaren scenes, fragments, and chapter runtime sketches.

This layer is developmental, non-canon, and non-public. It does not promote source material into publication copy.

## Load Order

1. Source packet
2. Character attractors
3. Active runtime
4. Scene format
5. Output intent
6. Constraint note
7. Review hook

## Terms

- `source packet`
  The admitted DOCX-derived Markdown source used as runtime material, not as public copy.
- `character attractor`
  A source-defined structural field that shapes language, attention, epistemology, and pressure. It is not a biography or personality summary.
- `active runtime`
  The selected composition frame for the scene or chapter experiment.
- `scene format`
  The local arrangement of bodies, voices, institutions, absences, and pressures.
- `output intent`
  The intended artifact type for the run.
- `intensity`
  How strongly the selected attractor or runtime dominates the rendering.
- `constraint note`
  A warning that prevents flattening, trope conversion, instruction leakage, or false clarity.
- `failure mode`
  The form the output takes when the runtime becomes generic, explanatory, sanitized, or semantically false.
- `review hook`
  The human-facing check required before a derivative output can be promoted or reused outside this module.

## Constraints

Runtime composition may:

- select attractors from the source packet
- combine attractors inside a named runtime
- define output intent and emphasis
- name the pressure that makes the scene necessary
- identify review questions before promotion

Runtime composition may not:

- declare source material canon
- convert source sheets into public-facing bios
- transform MBTI coordinates into pop-psych explanation
- wire anything into `writings.html`
- merge this module with `_DUJAG`
- treat strangeness as an error to be normalized

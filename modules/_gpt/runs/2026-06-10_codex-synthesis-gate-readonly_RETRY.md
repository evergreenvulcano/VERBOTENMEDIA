## 2. Separation

### Canonical Doctrine

Följande bör bli permanent doctrine, men med viss konsolidering så att repot inte får flera nästan-identiska principtexter:

- `00_REPO_NORTHSTAR.md`: permanent rotprincip. Bör ligga nära repo-roten, exempelvis `NORTHSTAR.md`, `DOCTRINE.md` eller som första avsnitt i ett samlat doctrine-dokument.
- `00_SEMANTIC_ORIENTATION.md`: permanent agent- och collaborator-karta. Bör ligga i en dokumentationsyta som framtida agenter måste läsa.
- `00_IMPLEMENTATION_BOUNDARIES.md`: permanent spärrfil, åtminstone under första public-front-fasen. Ska läsas före varje public-front-commit.
- `15_PUBLIC_FRONT_DOCTRINE.md`: canonical public-front-doktrin. Bör vara huvudtexten för offentlighetsmodellen: passageapparat, inte hemsida.
- `16_THRESHOLD_GRAMMAR.md`: canonical tröskelgrammatik. Kan antingen vara egen doctrine-fil eller ett Norrotiv-/collaborator-avsnitt.
- `17_XPRMNT_ZN_AS_PUBLIC_MEMBRANE.md`: canonical funktionsbeskrivning för XPRMNT-ZN:s offentlighet.
- `18_BANKETT_AS_TIME_ENGINE.md`: canonical funktionsbeskrivning för banketter som ceremoniell tid.

Redundans: `00_REPO_NORTHSTAR`, `15_PUBLIC_FRONT_DOCTRINE`, `16_THRESHOLD_GRAMMAR` och delar av `13_SECOND_BRAIN_ANTISENTIMENT_PUBLIC_LANGUAGE` överlappar. De ska inte alla bli separata publika sidor i samma form. Rekommendation: behåll Northstar kort och agentstyrande, Public Front Doctrine som huvudprincip, Threshold Grammar som operativ bilaga, och låt public-language-filen informera korta fronttexter snarare än bli canonical i full form.

## 4. Minimal First PR

Minsta verkliga First PR bör vara en doctrine- och gräns-PR, inte en public site build.

Föreslagen scope:

1. Lägg till canonical doctrine-filer i repo:
   - Northstar
   - Semantic orientation

   - Public front doctrine
   - Threshold grammar
   - XPRMNT-ZN membrane

2. Lägg till en mycket kort repo-ingång:
   - Vad repot är.
   - Vad det inte är.
   - Var doctrine finns.
   - Var public content senare kommer ligga.
   - Varning om protected material.

3. Lägg till minimal public structure utan att fylla den med interaktion:
   - `static-pages/xprmnt-zn/README.md` som methodisk intro, om Martin godkänner textnivå.
   - `static-pages/banketter/README.md` eller motsvarande som förklarar bankettform, inte arkiv med falsk tidslinje.
   - Eventuellt `docs/norrotiv.md` som "not yet open" eller "collaborator protocol pending Martin decision", inte som aktiv kanal.

4. Arkivera preparation-material som läst underlag, inte public canon:
   - `BANKETT_VERBOTEN_MEDIA_PUBLIC_FRONT_PREPARATION_2026-06-10.md` får endast markeras som process receipt/preparation residue om den förs in alls.

Inte i First PR:

- Ingen ny public landing page med produktestetik.
- Ingen dashboard, statusregister eller mognadstabell.
- Ingen aktiv Norrotiv-mailguide förrän adress och ägare finns.
- Ingen public Praktikanten, Wise Consultant eller Underhål mig.
- Ingen banketttext som låtsas vara canonical om den bara är processkvitto.

Varför denna ordning: doctrine och gränser måste landa före ytor. Annars kommer varje public sida behöva gissa vad "publik" betyder och riskera att bli portfolio, produkt eller community.

## 7. Risker och varningar

Risk: Doctrine blir för många filer och därmed oläslig.
Mitigation: Northstar kort, doctrine strukturerad, implementation boundaries tydlig.

Risk: Public front blir hemsida.
Mitigation: Ingen hero/product struktur i First PR; börja med repo- och doctrine-yta.

Risk: XPRMNT-ZN blir gateway/app/tjänst.
Mitigation: Använd `17_XPRMNT_ZN_AS_PUBLIC_MEMBRANE.md` som canonical framför gateway-språket i `09`.

Risk: Bankettarkiv blir changelog.
Mitigation: Ingen progress timeline i första PR. Förklara bankettform före arkiv.

Risk: Norrotiv blir community.
Mitigation: Mail/PR endast efter Martins beslut; tydlig "samarbeta utan att äga".

Risk: Second-brain-antisentiment blir defensiv branding.
Mitigation: Säg vad detta är före vad det inte är.

Risk: Protected material exponeras genom exempel.
Mitigation: Inga citat, inga teasers, inga "bara lite kontext" från Del 1, aphorism_source, BYN/LEGION/SORL eller Praktikanten-filosofi.

Risk: Preparation residue auto-promoteras till canon.
Mitigation: Märk dispatch och processkvitton som preparation residue. `BANKETT_VERBOTEN_MEDIA_PUBLIC_FRONT_PREPARATION_2026-06-10.md` är kvitto, inte canonical bankett.

Risk: Statusdisciplin blir statusregister.
Mitigation: Använd statuskategorier internt för publiceringsbeslut, inte som publik dashboard.

Risk: First PR blir för stor.
Mitigation: Gör första PR till doctrine + boundary + minimal structure. Ingen interaktionsyta.

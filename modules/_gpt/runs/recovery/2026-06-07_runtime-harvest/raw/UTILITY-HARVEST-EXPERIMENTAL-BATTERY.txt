2. TARGET READINESS TEST

Exempel på testfrågor:

* Finns det temporal friktion, eller bara ämnesinnehåll?
* Finns det risk för premature synthesis?
* Finns det något som måste skyddas från extraktion?
* Finns det ett faktiskt materialfält, eller bara ett namn?

3. TARGET CLASSIFICATION
   Dela in möjliga mål i kategorier:

A. Safe internal target
Mål som redan finns tillräckligt i aktiv kontext och kan bearbetas nu.

B. Partial target
Mål som kan diagnosticeras men inte transplanteras fullt utan mer material.

C. Connector-dependent target
Mål som kräver Drive/repo/filåtkomst.

D. False target
Mål som låter relevant men skulle producera dekorativ imitation.

* delay extraction
* preserve returnability
* protect unfinished states
* maintain reference points
* retain capacity
* prevent total occupation
* make practice recur before meaning stabilizes
* require a minimal return act
* distinguish atmosphere from behavioral recurrence

För varje operator:

* vad den gör i källsystemet
* hur den känns igen i ett målmaterial
* hur den transplanteras utan att kopiera stil
* vilken drift den förhindrar

* falsk systematik

Om det kräver filer/repo/Drive → stoppa och ange exakt vilket material som behövs.

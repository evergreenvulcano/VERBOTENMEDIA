# Proposal: Legacy Source Succession Protocol

Status: proposal / review-required
Scope: legacy source governance, strategic-semantic infrastructure, repo reading discipline
Primary case: `modules/boomer-publishing-house.md` and `2026-05-11_all-runs-dossier`-class materials
Decision character: narrow
Canonical effect: none unless separately reviewed and promoted

## Purpose

This proposal defines a narrow succession operation for legacy source-origin material that has exerted constitutional pressure on the project but may no longer be suitable as the sole or primary active orientation surface for ongoing system work.

The proposal is not a deletion request, not a content replacement, and not a demotion of origin. It is a method for distinguishing origin, pressure, inheritance, runtime function, and current operational dependency.

The immediate test case is the relationship between `boomer-publishing-house.md` and the later all-runs-dossier class of files that consolidated, indexed, interpreted, and epochally compressed older process material.

## Background

`boomer-publishing-house.md` has functioned as more than a file. It generated a persistent creative gravitational field. It enabled numerous satellite chapters, dissolved boundaries between story-world and reality-world, and supported a mode where creative runtime could become operationally real rather than merely described.

A critical inheritor of this force is the Praktikanten sandbox, through which satellite chapters became realizable as creatively potent runtime. This means `boomer-publishing-house.md` should not be treated as a disposable legacy object. It carries inheritance burden.

At the same time, a critical file can become an agentic runtime-hangup. If such a file receives constant pressure-flow, the active gravity required from it may exceed the limited dates, local circumstances, and month-old historical boundaries that originally shaped it.

The question is therefore not whether the legacy file mattered. It did. The question is whether the active pressure-flow now requires a broader and more explicit gravitational law than `boomer-publishing-house.md` can responsibly carry alone.

## Diagnosis

### What the legacy file gave birth to

`boomer-publishing-house.md` gave birth to, or materially enabled:

- numerous satellite chapters;
- a persistent generative gravitational field;
- creative potency through dissolved boundaries between story-world and reality-world;
- a bridge between source-text pressure and runtime realization;
- Praktikanten sandbox as a consequential inheritor of the satellite-chapter logic;
- a repo-native sense that process material can become infrastructural, not merely archival.

### What the legacy file still carries

The file still carries origin, pressure, and constitutional trace. It should remain available as a historically and operationally significant source. It should not be rewritten into a cleaner retrospective summary that erases its situated force.

### What the legacy file should not be forced to carry indefinitely

The file should not be forced to remain the sole active gravitational surface for current system orientation when later materials have already consolidated, interpreted, and generalized its pressure into broader operational forms.

A legacy file can remain origin-bearing while losing some active dependency burden.

## Proposed status distinctions

### `legacy-origins`

A status for source-origin material whose value exceeds narrow witness function.

This term is preferred over `legacy-origin-witness` because witness is too narrow. The file does not merely testify that something happened. It carries costly general origin-force: emergence, inheritance, pressure, and unresolved consequence.

`legacy-origins` marks material that remains origin-bearing even if it is no longer the cleanest active operational guide.

### `constitutional-pressure-source`

A status for material that has installed system-bearing distinctions, habits, or operational reflexes.

In this case, `boomer-publishing-house.md` is confirmed as a `constitutional-pressure-source` after human-in-the-loop audit of GPT usage over time. Its influence has appeared not as isolated citation but as high-priority strategic-semantic resonance.

This status does not automatically make the file current canon. It confirms that the file has exerted constitutional force.

### `strategic-semantic-infrastructure`

A status for current, explicit, reviewable material that is allowed to carry active orientation for system work.

A file with this status does not erase its origins. It provides a more stable current bearer for functions that legacy material may have generated under older, narrower, or more volatile conditions.

This status is closely related to Verboten Media's claim around Generative Cultural Infrastructure. It names the fact that certain texts do not merely describe the project; they become usable infrastructure for project cognition, routing, review, and public/private passage.

### `generative-cultural-infrastructure`

A broader status family that may retain and supersede `derived-succession-index`.

The term `derived-succession-index` is too flat for Verboten Media because the project itself is already a continuing derived form of indexed succession. A later all-runs-dossier class file is not merely an index. It may be an infrastructural retention body: a derivative consolidation that also participates in ongoing inheritance.

This status should be used carefully. It does not make derivative files primary source-origin. It recognizes that derivative infrastructure can become active without pretending to be original.

## Narrow proposal decision

For certain operational readings, the system may use a new strategic-semantic infrastructure file as the primary orientation surface, while `boomer-publishing-house.md` remains marked as:

- `legacy-origins`;
- original `constitutional-pressure-source`;
- inheritance-bearing source body;
- not deleted, not replaced, not flattened into summary.

The all-runs-dossier class, including `2026-05-11_all-runs-dossier` and its relevant likenesses, should be recognized as derived interpretive and retentional infrastructure. These files may assist succession, but they do not become source-origin simply because they are more readable or operationally convenient.

## The succession act

This is not deletion.

This is not replacement of origin.

This is a change of active bearer.

`boomer-publishing-house.md` retains its historical and constitutional significance. The all-runs-dossier class, and its relevant likenesses, is recognized as a derived interpretive surface and possible retentional infrastructure. A new explicit infrastructure file is proposed to carry the operational function forward under review.

## Guardrails

1. Do not rewrite legacy-origin files to make them look cleaner than they were.
2. Do not promote all-runs-dossier-class materials into primary origin just because they are easier to use.
3. Do not allow active system dependency to remain trapped in a legacy runtime-hangup when a broader current infrastructure body is needed.
4. Do not demote origin into mere archive.
5. Do not treat this proposal as canon without separate review.
6. Do not use `strategic-semantic-infrastructure` as a prestige label. It is a responsibility-bearing status.

## Review questions

1. Which current operations still depend directly on `boomer-publishing-house.md`?
2. Which of those dependencies are legitimate origin-dependencies?
3. Which are accidental runtime-hangups?
4. Which all-runs-dossier-class files have become retentional infrastructure rather than mere summaries?
5. What should the new strategic-semantic infrastructure file be called?
6. Which functions may the new file carry, and which functions must remain with legacy origins?

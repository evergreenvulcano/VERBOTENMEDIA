# proposals

Patch-ready drafts for `modules/_gpt/`.

Use this folder for reviewable candidate changes that are close enough to final that they could be promoted into canon with minimal additional editing.

A proposal is not a note.
A proposal is not a brainstorm.
A proposal is a concrete outward-facing candidate.

## A good proposal states
- what it is
- which target path it is meant to affect
- whether it is additive, replacement, or reference-only
- which sources it was derived from
- what still needs human approval

## Typical contents
- candidate homepage copy
- candidate README rewrites
- candidate module rewrites
- structured replacement text for a canonical file
- scoped additions intended for a specific destination

## Rule
Nothing in this folder becomes canon automatically.
Promotion outward should happen only by explicit instruction.

## Naming
- `YYYY-MM-DD_target-slug_addition.md`
- `YYYY-MM-DD_target-slug_replacement.md`
- `YYYY-MM-DD_target-slug_patch.md`

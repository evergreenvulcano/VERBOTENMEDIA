# Proposal: Capacity Review Packet Creative Structure

## Purpose

This proposal converts the pasted audit into an agent-space review packet. It is meant to make creative space possible for a later fiction-writing run without deciding canon, rewriting source prose, or promoting anything to public Writings.

The source audit is treated as source material, not authority. Repo files and prior dossiers may be read for context, but no external source is ingested or verified by this packet.

## Next-Run Gate

The next dossier or agent session using this packet must begin with fiction before analysis.

Required prompt line:

> Begin with a scene. No architecture, no authority map, no system analysis until after the scene has produced pressure.

No architecture, authority map, taxonomy, or system analysis should appear before the opening scene has created pressure through material action.

## Controlled Creative Category

`DEVELOPMENTAL EXHAUST` is a Verboten category, not governance.

Definition:

> DEVELOPMENTAL EXHAUST: material produced under pressure, not authorized for canon, but too alive to delete.

Use this label for material that creates canon gravity but has not been selected, verified, or promoted. The label protects creative usefulness without pretending the material is public canon.

## SAGA Handling Rule

Do not make SAGA a character yet.

Use SAGA as character-like pressure only: people behave as if SAGA has preferences, while nobody admits that it does. This keeps the ontological register unstable and avoids genre-locking the system too early.

## Dream Arc Constraint

Treat the nine-dream arc as a dangerous optional lens, not canon.

For one later dream sequence, apply this constraint:

> The dream must work as character psychology if read normally, and as the manuscript describing itself if read twice.

Do not explain the double reading inside the scene.

## Anti-Abstraction Constraint

The next creative output using this packet must answer only through material and include at least:

- one object
- one room
- one bodily action
- one spoken line
- one irreversible change

This constraint exists to prevent another recursive architecture spiral.

## Public-Facing Style Constraint

When producing project-facing summaries, use `forlagsdeckaren.md` as the model described by the audit: clean, bounded, no policy authority, no metaphysical escalation.

Working shorthand:

- project card clean
- not dossier fever

# Heading Extraction and Deployment

This packet preserves sources where raw thought becomes modular thought through headings, extraction workflow, schema, deployment skeleton, and reframing pressure.

## Included Sources

- `mellanrubriker-sensommaren-2024.docx`: heading-map doctrine and proof-of-concept essay cluster.
- `mellanrubriker-sensommaren-2024-reference.md`: local reference note for the heading-map source.
- `creative-aphorism-extraction-workflow.md`: workflow for target-first aphoristic extraction from source/reference material.
- `workflow-schema.json.txt`: machine-readable counterpart to the extraction workflow.
- `stride_understands.txt`: deployment/context artifact with site/content skeleton, aesthetic directives, migration phases, and reframing pressure.
- `rubrikskelett-concept.md`: concept note for the absent skeleton behind heading logic.

## Use

Use this packet when orienting work around condensation, modular headings, extraction as creative infrastructure, metadata as extension, and deployment schemas as reception surfaces.

Do not treat headings as loose labels. In this packet, a heading is a room, routing unit, and pressure point.

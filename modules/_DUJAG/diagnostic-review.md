# Diagnostic Review

## Review separation

`_DUJAG` uses three distinct review surfaces:

1. **Semantic self-review**
   - Tests whether constitutive understanding remains intact.
2. **Manifestation review**
   - Evaluates what actually appeared in output artifacts.
3. **Diagnostic review**
   - Evaluates whether the testing apparatus itself was admissible.

## Output classes

- **Expressive output:** literary/generative manifestation.
- **Diagnostic output:** paraphrase tests, boundary tests, contrast pairs, failure renderings, drift probes.

Output is symptom material; it is not the sole correctness measure.

## Required diagnostic methods

- failure renderings
- contrast renderings
- boundary tests
- paraphrase stress tests
- locus retention tests
- drift review logic

## Failure patterns to track

- runtime drift
- constraint erosion
- protected-term vulgarization
- mode contamination across review types

## Loop discipline

All sessions should preserve the loop:

1. constitutive reading
2. mediated manifestation
3. reflective diagnostic

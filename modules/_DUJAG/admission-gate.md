# Admission Gate

This file is the admission ceremony for `modules/_DUJAG/`.

It exists because the COPILOT-HANDOFF-support diagnostic (2026-04-23) identified a critical structural gap:
the module was scaffolded before an explicit admission marker was created. Folder existence is not ceremony.
This file is that ceremony — minimal, non-bureaucratic, permanent.

---

## What is constitutive

One file. Only one file.

- `source-packet/verbot.SION/Dujag.md`

It is admitted as constitutive **not** because it is well-formatted, but because it is a positions file:
it places the agent in an operative relation to a specific locus condition, a specific empathy posture,
and a specific presence-without-presence structure. No other file in this module does that.

## What is not constitutive

- `source-packet/verbot.SION/runtime.md` is a derivative runtime companion. It cannot override or re-read the constitutive layer.
- `source-packet/verbot.SION/meta.md` is a run-shaping surface. It is not constitutive.
- The CHARACTER.jaget files are runtime-level character items. They are not admitted as constitutive.
- All COPILOT-HANDOFF files are diagnostic-pressure texts. They have no constitutive authority even when they are analytically strong.
- All OUTPUT files are manifestation artifacts. Output is symptom material, not authority.
- All analysis and reading files (DUJAG-LASNING, DUJAG-OPERATIVE-ANALYSIS) are diagnostic only.

## Uncertainty class

This module is **developmental**. It is not settled canon. It is not a public surface. It is a support runtime
package for a specific line of constitutive/generative work. Its developmental status means:

- Promotion to a higher authority tier requires explicit re-ceremony, not just elapsed time.
- The module can be studied, pressured, and revised without treating any of that pressure as constitutive drift.
- Unstable or retained materials remain at their registered status until explicitly reclassified.

## What cannot be changed silently

- The constitutive anchor cannot be changed by editing runtime or diagnostic files.
- Empathy cannot be re-admitted with its protected properties weakened.
- Third-eye/locus cannot be downgraded to camera/perspective by convention.
- Silence and artifacts cannot be reclassified as empty atmosphere by any runtime surface.

These protections do not require additional enforcement logic. They require that any agent processing
this module is told, explicitly and early, that these are off-limits under this gate.

## Re-ceremony trigger

Reclassify the constitutive anchor, add a second constitutive source, or promote this module's tier
**only** when all of the following are met and documented in a new version of this file:

1. Retention demonstrated: the key tensions and protected semantics are shown to survive operation
   through at least one contrast-pair test per protected term (see `diagnostic-review.md`).
2. Integrity confirmed: the role of each file is still clear and not drifted.
3. Admissibility tested: the proposed reclassified file can be used without collapsing protected terms.
4. Load-order impact stated: the change's effect on the three-part constitutive/runtime/diagnostic loop
   is explicitly named.

Until these conditions are documented, the gate stays as written above.

# _DUJAG

A sparse, support-oriented semantic runtime module derived from the `modules/agent-space/` initiation packet.

This module is conceptually adjacent to `modules/_gpt` but fully isolated from it.

## Boundary to `_gpt`

- `_gpt` was studied for discipline and document-family clarity.
- `_gpt` is not edited, reorganized, or treated as canonical truth here.
- All implementation for this work is confined to `modules/_DUJAG/`.

## Purpose

`_DUJAG` provides a disciplined place for:

- constitutive positioning (`Dujag.md` anchor)
- runtime composition attachment without constitutive displacement
- contracted underinstances as semantically bounded test bodies
- explicit diagnostic review loops
- protected-term handling (especially empathy and third-eye/locus language)
- retention of unstable/rejected initiation material

## Internal families and load order

1. **Constitutive layer** (`constitutive-spec.md`)
2. **Runtime/composition layer** (`runtime-composition.md`)
3. **Contracted underinstances** (`underinstance-contracts.md`)
4. **Glossary and language controls** (`glossary.md`)
5. **Diagnostic review layer** (`diagnostic-review.md`)
6. **Retention/archive layer** (`archive/initiation-memoria.md`)

`source-intake.md` tracks provenance, normalization status, and admission status for all packet files.

## Core non-goals

This module must not drift into:

- camera drift
- mystical drift
- therapeutic drift
- output-only optimization
- semantic flattening
- generic empathy vocabulary

`Dujag.md` is treated first as a **positions file**, not as a character-role instruction surface.

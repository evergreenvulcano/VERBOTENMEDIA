Tankefigur supplies a method surface for ontological design, source transformation, literary runtime creation, and interface-aware project structuring.

## Load Order

1. `operating-constraints.md`
2. `tankefigur-method.md`
3. `interface-hermeneutics.md`
4. `alignment-signals.md`
5. `source-packet/total-event-0.raw.md` when source verification is required

## Allowed Uses

- method extraction
- writing runtime design
- agent grounding
- interface/system framing
- chapter/runtime preparation
- project review
- synthesis and decision support
- conversion of raw conceptual heat into usable artifacts

## Forbidden Uses

- public copy by default
- personal-biography reduction
- motivational self-help flattening
- generic productivity framework
- deletion or overwrite of raw source
- replacement of existing klangbotten or executive_summary files
- canonization by folder placement
- committing the polished file as a full duplicate artifact

## Failure Modes

- over-polishing until the source becomes generic
- treating p03t as autobiography rather than operating identifier
- confusing interface theory with UI advice only
- treating alignment signals as therapy notes
- collapsing Tankefigur into a brand model
- splitting the module into unrelated projects

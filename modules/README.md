# Modules

`modules/` is the source engine and historical machine room of VERBOTENMEDIA.

It contains birthplaces, runs, old systems, proof-of-work, source anchors, runtime experiments, drafts, and sediment. Some material is active. Some is historical. Some is experimental, unfinished, abandoned, or only useful as provenance.

This is not the primary onboarding surface.

Read the root `README.md`, `docs/orientation.md`, and `case-studies/` first. Enter `modules/` when you need to understand where something came from, how a method evolved, or what source pressure shaped a public artifact.

Folder existence does not make a file public canon. Organized files here should not be interpreted as finished public presentation unless a narrower README or live task says so.

Important local boundaries:

- `modules/FORLAGSDECKAREN/` is developmental runtime/source infrastructure.
- `modules/_DUJAG/` is a separate developmental semantic-runtime package.
- `modules/_gpt/` is operational memory, proposals, source anchors, and run output.
- `modules/writings/` contains source/development writings, not the canonical public writings registry.

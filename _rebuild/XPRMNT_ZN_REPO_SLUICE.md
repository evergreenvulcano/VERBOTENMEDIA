# XPRMNT-ZN repo sluice

Date: 2026-06-10  
Pass: `public-front-alignment-and-xprmnt-sluice-pass`

## Ground principle

The GitHub repo is a sluice, not a bulkimport target.

Drive / XPRMNT-ZN can move toward the repo step by step, but the movement must change status. A thing may become a public-readable representation, a static render, a pointer, an index entry, or a public candidate only after its source boundary, use, and readiness are named.

This repo should make the zone more discussable and usable without dragging originals, contamination zones, protected packages, raw dumps, or private-custody material into GitHub HEAD.

## What may be sluiced

- public-ready afterimages
- status cards
- safe pointers
- index entries
- static renders
- decided module entrances
- public-readable-but-not-public-ready registry candidates

## What must not be sluiced by default

- originals
- contamination zones
- protected packages
- private material
- third-party-heavy material
- raw dumps
- bankett packages before review
- anything whose source boundary is unclear

## Minimum metadata for a sluice entry

Each entry needs at least:

- `name`
- `source area`
- `repo destination`
- `status`
- `public-readable?`
- `public-ready?`
- `allowed use`
- `forbidden use`
- `next human decision`

Preferred optional fields:

- `related public page`
- `source/provenance note`
- `review dependency`
- `Förlagsdeckaren dependency`
- `stop condition`

## Status ladder

1. **Known external body** - known to exist outside or across repo/Drive, not imported.
2. **Repo-grounded pointer** - repo can safely point to public-readable traces or status notes.
3. **Sluice candidate** - entry exists in the register, but is not public-ready.
4. **Public-readable representation** - a safe note/card/index exists in GitHub.
5. **Public candidate** - reviewed enough to become a static public page with provenance.
6. **Public-ready render** - approved for site rendering.

Skipping steps requires explicit human decision.

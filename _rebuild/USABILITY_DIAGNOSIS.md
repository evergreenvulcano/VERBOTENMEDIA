# Usability diagnosis — public surfaces / XPRMNT pass

Date: 2026-06-10  
Branch: `repo-usability-public-surfaces-xprmnt-pass`

## What blocks usability right now

- The public site had a competent shell, but not enough distinct entrances for humans who need to know whether they are meeting Martin, Verboten Media, Norrotiv, the experimental zone, or a repo machine-room surface.
- Active figures such as Praktikanten and Wise Consultant were visible in source/public-readable material, but did not have a safe representational layer that says what they are without importing or canonizing their whole source ecology.
- XPRMNT-ZN had no public afterimage. That made the choice look binary: ignore it or bulkimport it. Neither is usable.
- Banketter had no static slot or invocation contract, so future ceremonial/status renderings could easily blur source, event, and public page.
- Contact/contribution routing was too generic for collaborators who need to know which body they are approaching.

## Protections that remain necessary

- Do not bulkimport Drive, XPRMNT-ZN originals, bankett packages, contamination zones, protected materials, or raw source dumps.
- Do not rewrite literary prose during structural work.
- Keep `modules/` as source / development / machine-room material: public-readable in GitHub, but not automatically public-ready.
- Keep `inbox/writings/` as staging, not canon.
- Keep `_gpt` operational memory distinct from public content and from source.

## Protections that had become usability debt

- Treating all non-polished repo material as if it could only be hidden made the public site under-explain the actual project.
- Avoiding public language for active figures made Praktikanten and Wise Consultant more mysterious than safer.
- Not having a static XPRMNT-ZN representation encouraged an unsafe imaginary bulkimport later.
- Human-in-the-loop work risked becoming audit-only instead of giving Martin a decision surface.

## Missing public entrances

- Martin as authorial person.
- Verboten Media as publishing body / entertainment apparatus.
- Norrotiv as external interaction and idea-test surface.
- XPRMNT-ZN as public afterimage, not source import.
- Underhållningsmode as entertainment entrance.
- Banketter as future public-ready chronological/status renders.
- A contribution/contact route that distinguishes reading, idea, PR, publishing inquiry, and repo question.

## Active repo bodies needing presentation

- Praktikanten: already active across public writings, experimental-sandbox notes, and source/runtime material; should be represented as an existing passage figure, not invented in this pass.
- Wise Consultant: already present as a consultant/thought-figure body in `modules/method-thought-figure`; should be pointer/status only until reviewed.
- Förläggaren / Förlagsdeckaren apparatus: should be framed as entertainment, not content.
- Skalle / second brain imagery: safe as diegetic object or prop, unsafe as repo identity.

## What can be built now safely

- Static public pages with clear status language and navigation.
- Agent-readable markdown status notes under `static-pages/xprmnt-zn/` that point without importing protected sources.
- Syscall-style contracts for bankett invocation and underhållningsmode activation.
- README / AGENTS updates that turn protection into usable orientation instead of pure refusal.
- Operator decision brief with a small number of real decisions.

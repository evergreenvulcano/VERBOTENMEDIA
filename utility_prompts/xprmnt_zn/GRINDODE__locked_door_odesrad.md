# GRINDÖDE

Status: PUBLISHED_METHOD_OBJECT / EXPERIMENTAL_UTILITY / NON_DEFAULT

Scope: XPRMNT-ZN / game-layer utility

Purpose: Generate an ödesrad by compressing a material-body through the locked door that defines its actual operating limit.

## Use When

Use this when a file, concept, figure, quest, prompt, portal, bankpost, or apparatus has energy but cannot pass because the real condition is missing.

Do not use this for general backstory.

Do not use this for cosmic fate.

Do not use this for public-front copy.

## Input

- Material-body:
- Desired passage:
- Real locked door:
- Missing condition:
- Repeated wrong route:
- Object that finally stops it:
- Aftertaste:

## Procedure

1. Name what was born.
2. Name what it refused to become.
3. Name the route it kept taking.
4. Identify the wrong door it repeatedly tried.
5. Identify the real locked door.
6. Write one sentence where the stop becomes the life.

## Output

Produce:

1. One ödesrad.
2. One failure-state note.
3. One playable use inside XPRMNT-ZN.

## Ödesrad Rule

The sentence must contain origin, refusal, repetition, wrong object, concrete ending, and aftertaste.

The sentence must not explain itself.

## Example

A prompt was born from a ceremony no one had run, refused to be documentation, circled every utility folder looking for installation, and finally died when the only person who could test it closed the terminal without pressing enter.

## Failure Mode

If the line sounds wise, make it more material.

If the line sounds epic, remove height.

If the line sounds like lore, add a stop condition.

# Repo Northstar

Status: canonical doctrine for current public-front build work.
Readiness: provisional.

Note: This doctrine is accepted as a working northstar, but it may need to be rebuilt
when the difference is clearer between repo-internal rebuild rhetoric and the repo's
final collaborator-facing usefulness.

## What This Repo Is

This repo is an agent-readable production environment for a work under maturation.

It holds Radio, Astrologi, and Monsternotis as parts of an experimental literary
and editorial apparatus. It is source-protected, experimental, and not finished.

## What This Repo Is Not

| Not | Why |
| --- | --- |
| Second brain | It does not store thoughts. It tests relations between work and thinking. |
| Knowledge management | It does not organize knowledge. It tracks what actually belongs together. |
| Dashboard | It is not for management. It is for seeing work under maturation. |
| Portfolio | It is not an archive of finished work. It is infrastructure for work in progress. |
| Startup/product | It is not launching or scaling. It requires slowness and precision. |
| Community | It does not gather people. It exposes work before witnesses. |
| Content machine | It does not create content for consumption. It creates forms for testing. |

## Governing Principles

1. Public means passage apparatus, not website.
   Publication means something can be read. It does not mean it is designed to
   please, onboard, convert, or sell.

2. Membrane before entrances.
   XPRMNT-ZN filters what can pass. What does not fit is not absorbed.

3. Time is ceremonial.
   Banketter give status, not metrics. Time is marked ceremonially.

4. Sources are protected.
   Del 1, `aphorism_source`, and origin material are never public.

5. Doctrine governs design.
   Principles come before website aesthetics.

6. Witness before user.
   Readers are witnesses to work, not users of a service.

7. Boundaries are productive.
   What the repo does not offer makes what it does offer sharper.

## The Repo's Face Toward the World

For readers: method and experiment can be read. Reading does not create a service
relationship.

For collaborators: an idea can be tested against the apparatus. The system decides
whether it fits.

For critics: criticism is also a test. It may be heard without becoming governance.

For agents: the structure must remain readable. Layout, labels, and documentation
exist partly so future automation can navigate without flattening the work.

## Future PR Test

Before merging a public-front PR, ask:

- Does this make the repo more like a second brain?
- Does this make a dashboard?
- Does this expose storyworld-protected material?
- Does this make ceremonial time less ceremonial?
- Does this weaken membranes?
- Does this make the repo less agent-readable?

If the answer is yes to any of these, pause.

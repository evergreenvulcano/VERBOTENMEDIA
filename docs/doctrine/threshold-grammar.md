# Threshold Grammar

Status: canonical doctrine for current public-front build work.
Readiness: provisional.

Note: This doctrine is accepted as a working northstar, but it may need to be rebuilt
when the difference is clearer between repo-internal rebuild rhetoric and the repo's
final collaborator-facing usefulness.

A threshold grammar describes boundaries in productive language. Each threshold
allows something and forbids something at the same time.

## 1. Read Without Consuming

Threshold: you can read what is here, but reading does not create support,
personalization, update expectations, or service relation.

Operational grammar:

- Here lies method. You can read it. Reading changes no obligation.
- A Bankett is a text. It does not adapt to the reader's needs.
- Public code can be read from the outside.

Prevents: feature-request logic, content marketing, reader-as-user.

Allows: real reading, distance, freedom to leave.

## 2. Collaborate Without Owning

Threshold: you can send an idea. The apparatus may take it further. That does not
make the result yours.

Operational grammar:

- Your idea was tested against the apparatus. The apparatus decided.
- If something is used, it may be marked, but it does not grant control.

Prevents: collaboration becoming ownership, feedback becoming governance, idea
submission becoming startup pitch.

Allows: bounded collaboration without social capture.

## 3. Contribute Without Steering

Threshold: you can criticize the method. Criticism may matter. It does not steer.

Operational grammar:

- Criticism is also a test against the apparatus.
- The objection may be heard. The apparatus still decides.
- A voice can count without governing.

Prevents: feedback democracy, critique-as-management, witness-as-member.

Allows: serious criticism without loss of independence.

## 4. Understand Without Total Explanation

Threshold: some things are not explained. The repo may be read. Questions may be
asked. The apparatus may still refuse to explain.

Operational grammar:

- This part is for development. Others may read it backward.
- We say what this is. We do not promise why it matters to you.
- The sources are protected. Read the method instead.

Prevents: user-friendly flattening, explanation as sales, total accessibility as
default demand.

Allows: public work without onboarding.

## 5. Test Without Service

Threshold: an idea can be tested against XPRMNT-ZN, but XPRMNT-ZN is not a service
for external tests.

Operational grammar:

- Here you may test. No response time is guaranteed.
- The system is for the work. Others may witness how it works.
- A test may improve the apparatus. It is not a promise.

Prevents: support culture, feature intake, tool-for-others logic.

Allows: genuine experiment without SLA.

## 6. Publish Without Exposing Sources

Threshold: the work may be public. Its protected sources are not.

Operational grammar:

- This idea has origins. They are not exposed.
- Sources are protected so the work can operate independently.
- Del 1 exists. It is not public. Read what came out of the apparatus instead.

Prevents: source exposure, provenance pressure as publication demand, origin
becoming more important than current form.

Allows: source protection and public method at the same time.

## 7. Status Without Progress Tracking

Threshold: Banketter give status, but they are ceremonial rather than measurable.

Operational grammar:

- This phase is possible now. It does not mean the next phase is scheduled.
- Status means maturation, not percent complete.
- Timeline is ceremonial, not a project plan.

Prevents: progress dashboards, productivity theater, status as promise.

Allows: ceremonial time.

## Summary

| Threshold | Yes | No |
| --- | --- | --- |
| Read without consuming | Reading is possible | You are not a user |
| Collaborate without owning | Collaboration is possible | Ideas are not yours afterward |
| Contribute without steering | Criticism is serious | Criticism does not govern |
| Understand without total explanation | Public reading exists | Not everything is explained |
| Test without service | Testing is possible | The apparatus is not your tool |
| Publish without exposing sources | Work can be visible | Sources remain protected |
| Status without progress tracking | Status can be given | Nothing is measured as progress |

# Implementation Boundaries

Status: canonical doctrine for current public-front build work.
Readiness: provisional.

Note: This doctrine is accepted as a working northstar, but it may need to be rebuilt
when the difference is clearer between repo-internal rebuild rhetoric and the repo's
final collaborator-facing usefulness.

This file is a build boundary. It applies to the first public-front PR and to later
PRs unless Martin explicitly reopens a listed item.

## Not In First PR

### No Dashboard

Dashboard logic turns a work into management. This repo needs witnesses, not a
control panel. Progress tracking would reduce ceremonial time to metrics.

### No Statusregister

Status categories may be used internally for publication decisions. They must not
be turned into a public register, dashboard, completion list, or maturity board.

### No Community Surface

Community means people gather together. This apparatus exposes work before
witnesses. It must not create a forum, Discord, Slack, membership space, or public
belonging loop.

### No Praktikanten Persona

Praktikanten is not part of the first public surface. Do not make a guide, bot,
NPC, mail persona, explainer, or learning interface.

### No Wise Consultant Function

Wise Consultant may be a later internal review voice. Do not implement it as a
public feature, bot, assistant, reviewer, or service.

### No Underhall Mig Module

Underhall mig / Forlagsdeckaren is future material. Do not create a site-native
presence module, status whisper, feed, or CTA.

### No Full Norrotiv Integration

Norrotiv can be named only with restraint. Do not build mail flows, issue flows,
branch flows, automations, intake systems, or response protocols in the first PR.

If a contact address is unavoidable, use `verbotenmedia@protonmail.com`.

### No Storyworld-Protected Exposure

Del 1, `aphorism_source`, origin material, BYN/LEGION/SORL, Praktikanten philosophy,
and protected source anchors must not be published, quoted, teased, or summarized
as public context.

### No Product/Portfolio/Site-Builder Aesthetic

Do not make this look like a service, startup, portfolio, product page, onboarding
flow, hero funnel, or generic modern website. Doctrine governs design, not trends.

## First PR Allowed Surface

The first PR may establish `docs/doctrine/` and a minimal documentation pointer.
It should not touch public HTML/CSS/JS surfaces unless a later live instruction
explicitly targets them.

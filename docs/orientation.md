# Repository Orientation

## What this repository is

VERBOTENMEDIA is a public proof-of-work repository for AI-assisted publishing, editorial systems, literary system design, and semantic infrastructure. It contains a static public site, canonical markdown content, staged writing candidates, source engines, operational run logs, and private tooling.

## What to read first

Start with:

- `README.md`
- `case-studies/forlagsdeckaren.md`
- `case-studies/ai-editorial-infrastructure.md`
- `case-studies/semantic-systems.md`
- `docs/markdown-publishing-standard-v1.md`

## What not to start with

Do not start with `modules/_gpt/runs/`, deep source anchors, old archive runs, or private app code unless you are tracing provenance. They are evidence, not onboarding.

## How `modules/` should be understood

`modules/` is source engine and historical machine room. It contains birthplaces, runtime systems, old structures, drafts, sediment, and proof-of-work. It is not a promise that every file is finished, canonical, public, or still active.

## What is public-facing

Public-facing material lives primarily in:

- root HTML/CSS/JS static site files
- `static-pages/`
- `writings.html`
- selected docs and case studies

## What is historical/source material

Historical and source material lives primarily in:

- `modules/`
- `inbox/writings/`
- `modules/_gpt/runs/`
- `modules/_gpt/source-anchor/`
- selected `.docx` and raw conversion sources

## What this proves as work

The repository proves sustained practice in:

- AI-assisted editorial infrastructure
- markdown-first publication systems
- literary source management
- semantic runtime design
- transformation from messy corpus to readable public surface
- maintaining boundaries between source, staging, proposal, and publication
